## How to run the test suite

Prerequisites: Node 20+, `npm install` completed.

```bash
# From the repo root
npm run test:run
```

Expected output: **14 test suites, 204 tests, 0 failures**.

To run a single suite:

```bash
npx vitest run lib/budget/status.test.ts
```

To run lint and build checks (NFR-DX-01 gate):

```bash
npm run lint
npm run test:run
npm run build
```

All three must exit 0. On a clean checkout this takes under 60 seconds.

## Known gaps

**No Playwright recordings (TC-STACK-04).** The stack decision recorded in `docs/adr/` and `docs/requirements.md` (TC-STACK-04) explicitly excludes Playwright from the MVP. No `.webm` or `.png` recording artifacts exist. All FRs that require browser-visible behaviour (FR-SHELL-02 responsive layout, FR-TX-02 immediate list update, FR-BUDGET-02 progress bars, FR-CHART-04 lazy skeleton, FR-CHART-05 tooltips) are covered by manual test plan steps only.

**No automated a11y scan.** NFR-A11Y-01 (Lighthouse Accessibility >= 95) and NFR-A11Y-02 (WCAG AA contrast) are not machine-verified. The acceptance report marks these as post-MVP items.

**@trace annotation warnings.** `node scripts/check-traceability.mjs` reports 29 test-trace warnings because the `@trace` annotations in the test files use `FR-*` IDs that the traceability checker's regex did not match at the time the report was generated. The annotations are present in every test file header; the RTM documents this explicitly. The traceability check exits PASS (no failures).

**recording-evidence warnings.** All 29 FRs show a recording-evidence warning in the automated report because no recording manifest exists. This is expected given TC-STACK-04.
